# 为项目创建使用说明 README

## Goal

为 ds-register (OpenAI 账号池编排器) 项目编写一份完整、清晰的使用说明文档,帮助用户快速理解项目功能、安装部署和使用方法。

## What I already know

**项目基本信息:**
- 项目名称: OpenAI 账号池编排器 (ds-register)
- 版本: 2.0.0
- 许可证: MIT
- Python 版本: >= 3.10

**核心功能:**
- OpenAI 和 DeepSeek 账号自动注册
- Token 管理与健康检查
- Sub2Api 账号池维护与清理
- 多邮箱提供商支持
- 代理池集成
- Web UI 和 CLI 两种运行模式
- 实时日志推送 (SSE)

**技术栈:**
- FastAPI + Granian (后端服务)
- 原生 HTML/JS/CSS (前端 UI)
- curl_cffi (HTTP 客户端)
- loguru (日志系统)

**项目结构:**
- 核心代码: `core/`
- 前端静态文件: `static/`
- 运行时数据: `data/`
- 启动入口: `main.py`, `core/__main__.py`

**已有内容:**
- 当前 README.md 仅有一行标题 "# ds-register"
- pyproject.toml 包含项目元数据和依赖
- CLAUDE.md 包含开发指南和代码规范
- 示例脚本: `example_deepseek.py`

## Assumptions (temporary)

- 目标用户是有一定技术背景的开发者或运维人员
- 用户已经了解 OpenAI API 和账号池的基本概念
- 用户能够自行准备邮箱和代理资源
- README 使用中文编写 (根据项目现有文档风格)

## Decision (ADR-lite)

**Context**: README 需要同时服务两类读者：想要快速使用工具的最终用户，以及想要深入理解/扩展项目的开发者。

**Decision**: 采用分层组织结构 —— 从快速开始（面向用户）到深度定制（面向开发者）。

**Consequences**: 
- 文档会有明显的层级划分，用户可以按需阅读
- 前半部分侧重实用性，后半部分侧重技术性
- 文档长度会比单一受众的 README 更长，但结构清晰便于导航

## Document Structure (Confirmed)

**章节组织：**
1. **简介** - 项目是什么，解决什么问题
2. **功能特性** - 核心功能列表
3. **快速开始** - 安装 → 启动 → 基本使用（用户可在此停止）
4. **配置说明** - 主要配置项和典型场景示例
5. **架构概览** - 项目结构、核心模块（开发者关注）
6. **高级用法** - CLI 参数、API、部署优化
7. **故障排查** - 常见问题和解决方案
8. **贡献指南** - 开发环境、测试、提交规范

## Configuration Documentation Strategy (Confirmed)

**采用典型场景配置示例方式：**
- 不列出所有配置项（避免文档过长）
- 提供 3-4 个典型场景示例：
  1. 最小配置（仅必填项）
  2. 完整配置（包含可选优化）
  3. 多邮箱提供商配置
  4. 代理池集成配置
- 在示例中添加注释说明关键配置项作用
- 指向运行时生成的 `data/sync_config.json` 获取完整配置
